# Release Notes: v3.20.2

**Released**: 2026-05-31
**Theme**: Consumer-Surface Delivery
**Upgrade**: version-bump only — no breaking changes

## Overview

v3.20.0 advertised three consumer-facing capabilities (C-F1/C-P1/C-P3) in its CHANGELOG, but they landed only on the **author surface** — verified **0/13** in the template repositories consumers actually pull. The release still passed green because no gate resolved "advertised" to "actually pullable." v3.20.2 closes that gap:

- **C-F1** — `/aget-propose-actions` presentation (Evidence column · ▶ Recommendation · ⚠ Decisions) — synced into 12 templates; **created** in `template-document-processor-AGET`.
- **C-P1** — `close_gate_check.py` close-gate conformance guard — added to all 13.
- **C-P3** — `check_structural_skill_frontmatter` (#1489) — added **additively** to each template's `health_check.py` (local checks preserved).

Delivered **0/13 → 13/13**, verified by the consumer's own check (`gh api ?ref=main`), not a local proxy. v3.20.1 was a canonical-only tag (tag-vs-HEAD fix, #1549), never reached consumers — "spent"; v3.20.2 is the consumer-surface completion. Tracking: gmelli/aget-aget#1551.

## Scope (Tier-1-only — deferrals stated honestly)

This is a **delivery** release, not a feature release. The structural recurrence-prevention work is **deferred** rather than rushed:

- **Tier 2** — blocking consumer-reachability release gate (resolves every advertised capability to the consumer surface at release time). **Priority fast-follow.**
- **Tier 3** — canonical skill source-of-truth + skill divergence gating.
- `template-document-processor-AGET` `ci.yml` + v3.0 structural completion (adding CI would expose a pre-existing red).
