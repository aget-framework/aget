# v3.25.0 — Grounded Entities & Trusted Releases

**Released**: 2026-07-04 · **Type**: minor · **Theme**: balanced double-bill — a capability pole and an integrity pole, equal billing.

## The narrative

Two threads that have run through recent cycles converge here.

**Grounded Entities**: AGET's SKOS ontology (871 concepts) has historically been *decorative* — mentioned by specs, never normatively bound (L924 [instance-only per L600]: canonical specs carried zero inline `aget:concept/` URIs). v3.25.0 ships the first spec that binds vocabulary by URI as a normative act: **AGET_ENTITY_DIMENSION_SPEC v0.1.0** defines seven dimensions along which any domain entity is characterized (definition, interrelationship, action, lifecycle, persistence, governance, provenance), demonstrates the binding pattern in its own text, and arrives with its conformance validator (`validate_entity_dimension.py`, 16 V-tests, 12 executable). It is the reusable template the ontology-spec-binding initiative's remaining streams inherit. Alongside it, **aget-ask v1.0.0** graduates to production — a skill whose quality metric is entropy reduction on the next-action prediction, not thoroughness — and **AGET_FRICTION_SPEC v1.0.0** goes Active after genuine non-author forward-validation (a supervisor-run inherit/self-test/capture pass), with the triage-by-value gate that keeps healthy friction (push windows, authority-expanding edit prompts) from being "remediated" away.

**Trusted Releases**: v3.24.0 was declared RELEASED from visible markers while ~10 owed items fell through silently. v3.25.0 mechanizes the answer: **"RELEASED" is computed, never asserted.** The Definition-of-Done is a binary script; the close path recomputes it and blocks on RED; closure checklists are scanned for placeholder substance, not just ticked boxes; gate V-tests assert received-state (what the consumer observes) rather than producer-action ("pushed"); migration guidance must be reachable *at the tag*, not merely on HEAD. Fittingly, the first live run of the composed close-guard surfaced four real residual gaps in v3.24.0 itself — including the absence of this very file's v3.24.0 sibling, backfilled in this release.

## Self-application

The release-integrity tooling this cycle ships gated this cycle's own close (L922 [instance-only per L600]). The conformance model for templates changed to **reliance-only** (principal ruling D-1): a template's version label promises "runs the version's features"; specs are canonical-referenced via an `@aget-canonical-specs` line, never copied.
