# Release Notes: v3.20.3

**Released**: 2026-05-31
**Theme**: C-P3 Correctness Fix
**Upgrade**: version-bump only — no breaking changes (patch)

## Overview

v3.20.3 corrects a verifier defect shipped in v3.20.2 — **before any fleet rollout** (fix-once-not-N).

The C-P3 health check (`check_structural_skill_frontmatter`, #1489) skipped any D71-STRUCTURAL skill whose `SKILL.md` was absent (`if not sk.is_file(): continue`) and then reported **"clean,"** conflating *present-and-clean* with *absent, couldn't check*. An absent mandated skill is arguably the worse violation — the agent cannot model-invoke it — yet it was rendered green.

v3.20.3 makes the check honest: an absent D71-STRUCTURAL skill now emits a **warning** (`"N/4 present + clean; ABSENT: …"`), surfaced rather than hidden. Propagated to all 13 templates.

## How it was found (and why it mattered)

The defect surfaced during the **v3.20.2 supervisor pilot**: the supervisor's own `aget-close-project` is absent, so C-P3 reported "clean" having checked only 3 of 4 skills. Critically, a run-and-check-clean rollout pilot is **structurally blind** to a silent-skip — it would have PASSED a defective check and propagated it to the fleet. So the fix was made in canonical + 13 templates first; the fleet pulls the corrected C-P3 **once**, never inheriting the defect (gmelli/aget-aget#1553, CLOSED).

## Verification

- worker (missing 2 D71 skills) → **warning** listing the absent skills (was false-clean at v3.20.2).
- framework-AGET (all 4 present) → **clean** (correct).
- 13/13 templates carry the corrected C-P3 on the consumer surface; no regressions.
