# AGET v3.23.1 — Goal Tier (canonical) + Close-Authorization Guard

**Date**: 2026-06-20
**Type**: Minor (additive, no breaking changes)

## Summary

v3.23.1 is the fast-follow the v3.23.0 preview promised. It promotes the **Goal Tier to canonical** and ships the **close-authorization guard** that v3.23.0 explicitly deferred (and which a post-release correction confirmed was *not* in v3.23.0).

## What's in it

- **`AGET_GOAL_SPEC` v0.2.0 → canonical `specs/`.** A Goal is a durable, cross-session **outcome** — the governance tier between the singular North Star (one permanent purpose) and the Initiative (a body of work). The spec defines KAOS goal typing (Achieve / Maintain / Soft) and the load-bearing requirement that every Goal owns at least one regulating loop ⟨owner, trigger, review-action, consequence, cadence⟩, or it decays as an orphan.
- **`aget-create-goal` + `aget-propose-goals`.** The propose→commit verb pair, now with the spec canonical. Committed Goals are stored two-tier (committed on the governance surface; aspirational agent-internal).
- **`aget-close-project` + `close_authorization_guard.py`.** A close-time authorization guard wired into project close (Step 2.5): it blocks a close that asserts authorization without a linked authorizing event.

## Propagation

Version 3.23.0 → 3.23.1 across `aget/` + 13 templates. The three new skills were synced to all templates dependency-aware (each with its engine scripts), so the capability is available fleet-wide on upgrade.

## Deferred

The full Goal-Value scoring rubric is a future-release item; v3.23.1 minimally grounds the value dimension so the spec can reference it.
