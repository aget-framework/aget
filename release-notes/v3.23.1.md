# AGET v3.23.1

**Theme**: Goal Tier (canonical) + Close-Authorization Guard
**Date**: 2026-06-20
**Type**: Minor (additive, no breaking changes)

## Summary

The fast-follow the v3.23.0 preview promised: the Goal Tier lands in canonical, and the close-authorization guard (built but explicitly *not* shipped in v3.23.0) is now in `aget/` with template propagation.

## What's New

- **`AGET_GOAL_SPEC` v0.2.0 → canonical `specs/`.** A Goal is a durable, cross-session **outcome** — the governance tier between the singular North Star (one permanent purpose) and the Initiative (a body of work). Defines KAOS goal typing (Achieve / Maintain / Soft) and the requirement that every Goal owns at least one regulating loop ⟨owner, trigger, review-action, consequence, cadence⟩, or it decays as an orphan.
- **`aget-create-goal` + `aget-propose-goals`.** The propose→commit verb pair, now with the spec canonical. Committed Goals are stored two-tier (committed on the governance surface; aspirational agent-internal).
- **`aget-close-project` + `close_authorization_guard.py`.** A close-time authorization guard wired into project close (Step 2.5): it blocks a close that asserts authorization without a linked authorizing event.

## Compatibility

- **No breaking changes.** Additive only; existing skills/specs/scripts unchanged.
- Drop-in minor across `aget/` + 13 templates. Engine scripts are stdlib-only (import-smoke verified across archetypes).

## Propagation

Version 3.23.0 → 3.23.1 across `aget/` + 13 templates; the three new skills synced to all templates dependency-aware (each with its engine scripts), so the capability is available fleet-wide on upgrade.

## Deferred

The full Goal-Value scoring rubric is a future-release item; v3.23.1 minimally grounds the value dimension so the spec can reference it.
