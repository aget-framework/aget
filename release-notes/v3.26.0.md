# v3.26.0 — Signals & Contracts

**Released**: 2026-07-11 · **Type**: minor · **Theme**: the release hears its consumers (signals) and hardens what it promises them (contracts).

## The narrative

v3.25.0 shipped a week before this release, and within 48 hours its first consumer sweep surfaced roughly fifteen defects — including two payload bugs that had passed green through every framework gate. v3.26.0 is the release built from listening to that: nearly every item traces to a consumer-reported failure, a cross-seat convergence, or a contract gap the sweep exposed.

**Signals**: an agent should not have to discover staleness by accident. `wake_up.py` now reports at session start when the local framework version is behind the latest release — one line when behind, silence when current, fail-soft when offline. (Adopters of the earlier reference implementation should note: the `gh release view` form hangs indefinitely under a non-tty subprocess; the shipped form uses `gh api`. A fail-soft timeout can mask that hang so thoroughly the banner never fires — silence and health look identical, which is exactly the class the three-state contract below names.) `health_check.py` and `study_topic.py` gain extension hook points, ending the fork-or-halt dilemma a downstream fleet hit when local additions lived in base files that upgrades overwrite.

**Contracts**: several of this cycle's items are one idea applied at different layers — *a claim must be distinguishable from its absence*. The three-state check contract (PASS / FAIL / UNREACHABLE) makes "couldn't check" reportable, after a fleet-wide checker silently degraded for two releases. The terminal-state vocabulary makes "artifact created but not deployed" a representable plan state instead of a false COMPLETE. The study-topic search contract makes research output declare which surfaces it searched and which it deliberately did not. And the verify-before-claim coverage matrix generalizes it: verification is a matrix of claim-channels × enforcement-gates, filled asymmetrically across fleets — each seat had built the gate for the channel that last burned it, and stayed blind in the others. Two placements land now (pre-filing dedup/existence probes in `/aget-file-issue`; a has-it-run gate in `/aget-close-project` that blocks COMPLETE on execution-evidence-free mechanisms), one is spec-registered (conversational pre-assertion, CAP-SESSION-015), one is wired (lesson-first propagation at `/aget-record-lesson`).

**Release integrity continues its arc**: the tag-payload coherence gate ends the post-tag-repair-divergence class; the deprecation registry runs its first live rehearsal under the newly-adopted N-2 policy; and this release ships its own DEPLOYMENT_SPEC *at the tag* — closing the lapse its predecessor disclosed.

## Self-application

The contracts gated their author. The closer-mutates-scaffold rule's first enforcement sweep caught four placeholder survivors in its author's own just-closed plan. The permission-quality check's demonstration fixture is its author's own settings file (a bare write-capable grant, green under count-checking). The pre-filing probes' first live exemplar is the cycle's own retroactive tracker filing. The has-it-run gate will next be applied to the release plan that shipped it.
